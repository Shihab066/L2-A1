## 1. What are some differences between interfaces and types in TypeScript?

* টাইপ এর মাধ্যমে ইউনিয়ন, টাপল , প্রিমিটিভ টাইপ, অবজেক্ট, ফাংশন ডিফাইন করা যায়. ইন্টারফেস এর মাধ্যমে শুধু অবজেক্ট, ফাংশন টাইপ ডেফিনেশন করা যায়। 
* একই নামে একাধিক ইন্টারফেস তৈরি করা যায়, একই নাম হলে ইন্টারফেস অটোমেটিক্যালি মার্জ করে নেয়, কিন্তু টাইপ একই নামের হতে পারেনা একই নাম ব্যবহার করলে এরর দিবে। 

#### উদাহরণ
```
interface User {
  name: string;
}

interface User {
  age: number;
}

const u: User = {
  name: "Foysal",
  age: 25,
};

//এরর দিবে 
type User = {
  name: string;
}

type User = {
  age: number;
}
```
* ইন্টারফেস এক্সটেন্ড করতে এক্সটেন্ড কিওয়ার্ড ব্যবহার করা হয় .টাইপ এক্সটেন্ড করতে ইন্টারসেকশন অপারেটর এর ব্যবহার করতে হয়। 

#### উদাহরণ
```
interface A { x: number }
interface B extends A { y: number }

type C = { x: number }
type D = C & { y: number }
```

## 2. Explain the difference between any, unknown, and never types in TypeScript.
`. any` 
============================================

`any` ব্যবহার করলে টাইপস্ক্রিপ্ট আর টাইপ চেক করে না।\
মানে এখন যে কোনো টাইপ ব্যবহার করতে পারবো 

### উদাহরণ
```
let var: any = 5;

var = "hello";
var.toUpperCase();
var.push(1);
```
উপরের উদাহরণ-এ টাইপস্ক্রিপ্ট কোনো এরর দিবে না। কিন্তু এখানে দুইটা স্ট্রিং এবং এরে মেথড ব্যবহার করা হয়েছে, যদিও এটা স্ট্রিং এবং এরে না ও হতে পারে, তাই any টাইপ যতটা সম্ভব এড়িয়ে চলা উচিত। 

* * * * *

 `. unknown`
==============================================================

`unknown` হলো সেফ ভার্সন অব `any`

### উদাহরণ
```
let unknownVar: unknown = "hello";

unknownVar.toUpperCase(); // এরর দিবে দিবে কারণ টাইপস্ক্রিপ্ট জানেনা এইটা কোন টাইপ `

ব্যবহারের আগে টাইপ চেক করতে হবে:

if (typeof unknownVar === "string") {
  unknownVar.toUpperCase();
}
```

ব্যবহার:

-   যখন জানিনা ভ্যালু টাইপ কি কিন্তু `any` বা ভুল টাইপ ব্যবহার করতে চাইনা  


* * * * *

`. never`
=======================================

`never` এমন টাইপ যেটার কোনো ভ্যালু থাকতে পারে না

ব্যবহার:

যখন কোনো ফাংশন কোনো কিছু রিটার্ন করবেনা তখন রিটার্ন টাইপ `never` দেয়া যেতে পারে। 

### উদাহরণ
```
const error = (message: string): never => {
    throw new Error(message);
}
```
